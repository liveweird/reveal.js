Razor Generator

in multi-tier architecture

Environment

  • Modules have their own UIs
  • There are nonSPA pieces integrated into SPA experience
  • UI component libraries
  • Portable areas (non MVC contrib) in different projects

Razor Generator

  • Custom build tool
  • Generates cs files for views, layouts
  • Easy to register
  • Turns views into code, allowing to deploy classes compiled to assemblies only

How to (1):

  • Add RazorGenerator.MsBuild package
  • Define what will be compiled
    • 			<ItemGroup>
            <RazorSrcFiles Include="Views\**\*.cshtml" />
			</ItemGroup>
  • Build

How to (2):

  • In another lib, add package RazorGenerator.Mvc
  • and register usage.
    • 		var engine = new PrecompiledMvcEngine(
            typeof(PreApplicationStartCode).Assembly);
		ViewEngines.Engines.Add(engine);
		VirtualPathFactoryManager.RegisterVirtualPathFactory(engine);

Advantages

  • Turns views into code, allowing to deploy classes compiled to assemblies only
  • Let's make component projects!
  • Compiled artifacts - app's getting warm faster
