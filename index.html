Why Web Apps?

Web Apps pros

Light-weight (computing power & network bandwidth)

HTML5 + CSS3

Transparently deployable

OS agnostic

Doesn't require additional software / hardware

Mixable (easily integrated) on both client & server side

Customizable looks (styles)

Problems with Web Apps

Web Apps cons

Different browser = different runtime

Browser version lifecycle

Different rig = different resolution, different perf, ...

Presentation mixed with business logic

Security concerns

Crappy tooling (until quite recently)

JavaScript as a root of all evil

dynamic, loose type control

different engines (implementations)

very limited structuring support

The evolution of Web Apps

I. Let's pretend it's not Web (antique)

  1. HTML generated on the server side
  2. Server-side "controls"
  3. Heavy session (state)
  4. Thick client abstraction forced

II. HTTP restoration

  1. MVC. Rediscovered.
  2. Stateless by definition
  3. Based on bare HTTP
  4. Views composed directly of HTML elements, but ...
  5. ... in-view templating (server-side)

III. SPA

  1. JavaScript app libraries / frameworks
  2. Unmatched testability
  3. Reduced impact of connectivity issues
  4. High complexity
  5. Skill separation

Core SPA components

  1. Hash-bang client-side routing
  2. No post-backs
  3. Client-side data storage
  4. Heavy AJAX, relying on JSON data
  5. Templating in JavaScript
  6. SEO issues
